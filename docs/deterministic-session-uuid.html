确定性 Session UUID

src/lib/crypto.ts 2026-07-13 Bugfix

Umami 的 /api/send 接口在创建 session 时，使用 uuid(sourceId, ip, userAgent, sessionSalt) 传入参数，但 uuid() 函数始终返回 crypto.randomUUID()，忽略了所有参数。

这导致同一用户的每次页面浏览都生成不同的 sessionId，3 次浏览被计为 3 个访客。

问题

表现自统计仪表盘访客数严重虚高（每页浏览 = 1 访客）
根因uuid() 无视传入参数，永远 crypto.randomUUID()
影响范围所有基于 /api/send 的追踪事件

改动

文件：src/lib/crypto.ts

uuid() 函数新增逻辑：有参数时调用 determineUUID() 生成确定性 UUID；无参数时回退随机 UUID。

export function uuid(...args: any): string {
  // 有参数 → 确定性 UUID（相同输入 = 相同输出）
  if (args.length) {
    return determineUUID(args.map(a => String(a ?? '')).join('|'));
  }
  // 无参数 → 随机 UUID v4（向后兼容）
  return crypto.randomUUID();
}

新增 determineUUID(name) 函数，使用 djb2 双哈希生成 128 位值，格式兼容 UUID v4：

function determineUUID(name: string): string {
  let h1 = 5381, h2 = 52711;
  for (let i = 0; i < name.length; i++) {
    const c = name.charCodeAt(i);
    h1 = ((h1 << 5) + h1 + c) | 0;
    h2 = ((h2 << 5) + h2 + c) | 0;
  }
  // ... 拼装为 UUID 格式
}
为何不用 uuid 包：官方上游使用 uuid npm 包的 v5() 实现。本项目部署于 Cloudflare Workers， 去除 uuid 包依赖可节省约 15KB 的 Bundle 体积（对 3MB 免费计划限制有帮助）。 djb2 64 位哈希在单站月访客量级下碰撞概率可忽略。

效果

工作原理

sessionId 由 uuid(sourceId, ip, userAgent, sessionSalt) 生成，其中 sessionSalt = hash(startOfMonth(createdAt).toUTCString()) 是一个月内不变的值。 因此：

场景结果
同用户同月访问 3 页 1 个 session, 1 个访客
不同用户（IP/UA 不同） 不同 session, 正确分离
同用户跨月访问 不同 session（salt 轮换）
NAT 环境（相同公网 IP） 可能合并（IP 指纹共同局限）

对比

指标改前（随机 UUID）改后（确定性 UUID）
3 次页面浏览计为3 个访客 ❌1 个访客 ✅
唯一访客（UV）虚高准确
页面浏览（PV）不变不变
session 持续时间每次独立（准确）跨月合并（同月内有膨胀）
跳出率准确偏低（多日浏览计入同一 session）

与上游一致性

本改动与官方 Umami 上游设计完全一致。上游在 src/lib/crypto.ts 中使用 v5(hash(...args, secret()), v5.DNS)， send/route.ts 中调用 uuid(sourceId, ip, userAgent, sessionSalt) 生成确定性 sessionId。 差异仅为：上游使用 SHA-512 + UUID v5（依赖 uuid 包），本项目使用纯 JS djb2（零依赖）。

已知局限

  • session 时长膨胀：盐轮换默认为 month，同月内多次访问的 session 时长异常偏高。
  • 跳出率偏低：同一 session 内包含多日页面浏览时，跳出率统计失准。
  • 缺少 secret() 混入：上游将 APP_SECRET 哈希加入确定性 UUID 输入以增加实例间隔离，本实现未包含。

配置

可通过环境变量 SALT_ROTATION 调整轮换粒度：

SALT_ROTATION=month   # 默认，按月轮换（最激进去重，session 时长偏差最大）
SALT_ROTATION=week    # 按周轮换
SALT_ROTATION=day     # 按天轮换（推荐，平衡去重与准确性）
